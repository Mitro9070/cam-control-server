### 7.1 Слой ASCOM (`ASCOM.ProjectR1.Camera`, C#)

- Синхронизация охлаждения с Python API: `SetCCDTemperature` читает и после записи подтверждает `target_temp_c` из ответов `/camera/cooling/status` и `/camera/cooling/target`; после `Connect` вызывается подтягивание статуса охлаждения (`cooler_on`, целевая температура).
- `CoolerOn`: getter опрашивает сервер; при включении кулера запрос не навязывает фиксированный процент мощности (управление остаётся на стороне сервиса).
- `ReadoutMode` / `ReadoutModes`: дискретный набор скоростей, согласованный с `READOUT_SPEED_OPTIONS` в Python (`50 … 5000` kHz, подписи в UI).
- `Gain` / `GainMin` / `GainMax` / `Gains`: режимы 0/1 с человекочитаемыми подписями; запись через `PUT /settings` (при подключённой камере сервис применяет параметры к SDK).
- `SetupDialog()`: модальное окно Properties (`CameraSetupForm`) — выбор readout/gain, сохранение `ProfileId` в ASCOM Profile, `PUT /settings`, опционально `POST /camera/profiles/{id}/activate`.
- Vendor action `ProjectR1:WarmUp`: вызов `POST /camera/cooling/warmup` с параметрами по умолчанию (расширяемость для клиентов без отдельной ASCOM-команды warm-up).
- Версия драйвера в свойстве `DriverVersion` обновлена до **0.2**.

Затронутые файлы: `01_ASCOM_Local_Server/src/ASCOM.ProjectR1.Camera/CameraDriver.cs`, `CameraSetupForm.cs`, `ASCOM.ProjectR1.Camera.csproj` (подключены `System.Windows.Forms`, `System.Drawing`).

### 7.2 Слой Python camera-service

- `apply_sdk_imaging_settings()` в `camera_runtime`: при изменении настроек при активном подключении вызываются `set_readout_speed` и `set_sensor_output_mode` адаптера SDK.
- В `sdk_adapter` добавлены реализации этих методов для mock- и native-адаптера.
- `PUT /settings`: синхронизация `default_gain_mode` с `settings.sdk_sensor_output_mode` и попытка немедленного применения к SDK при `connected` (ошибки SDK логируются, сохранение настроек не откатывается).
- `_apply_runtime_settings_from_profile`: при активации профиля дополнительно выставляется `sdk_sensor_output_mode` из `gain_mode` профиля.

Затронутые файлы: `02_Python_Camera_Service/app/camera_runtime.py`, `sdk_adapter.py`, `app/api/routes.py`.
