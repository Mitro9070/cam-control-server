## 4. Ручной smoke test (быстрый)
1. Запустить Python service в режиме R1 (без обязательной БД).
3. Открыть UI, проверить health.
4. Выполнить connect.
5. Стартовать экспозицию 1 сек.
6. Проверить `ImageReady=true`.
7. Получить и проверить метаданные кадра.
8. Проверить сценарий warm-up.
9. Через ASCOM клиент повторить connect/exposure.
10. Выполнить disconnect.

Дополнительно для R1.1:
11. Подключить PostgreSQL.
12. Применить миграции.
13. Проверить запись событий в `event_log`.

Ожидаемый результат: сценарий проходит без критичных ошибок и без зависаний.
