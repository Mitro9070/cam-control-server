## 1) Что исправлено

### 1.1 Критичные исправления интеграции SDK/ASCOM
- Устранен сбой `JavaScriptSerializer maxJsonLength` в MaxIm DL:
  - в C# драйвере `ImageArray` переведен на бинарный endpoint `GET /api/v1/camera/images/latest/raw`;
  - JSON-путь с большим base64 для ASCOM-клиентов исключен из основного сценария.
- Исправлены ошибки `AbortExposure`/`StopExposure` в ASCOM при гонках завершения:
  - операции сделаны идемпотентными, ошибки «уже завершено» не валят клиент.
- Исправлены ошибки `CoolerOn=false` при transient `camera busy`:
  - выключение охлаждения сделано устойчивым к переходным состояниям SDK.
- Добавлена сериализация нативных вызовов SDK через lock + retry/backoff/unlock sequence:
  - снижены интермиттентные `camera busy` в сетевом bootstrap.

### 1.2 Исправления API/Runtime
- Добавлено поле `percent` в статус экспозиции.
- Добавлен совместимый endpoint warmup-статуса:
  - `GET /camera/cooling/warmup/{id}/status` (и сохранен legacy `/{id}`).
- Добавлен бинарный wire-format для кадров:
  - `GET /camera/images/latest/raw` + заголовки метаданных.
- Для native режима стабилизировано чтение кадра:
  - чтение через корректный буфер для полного кадра, чтобы исключить access violation на ROI-сценариях.
- Добавлено логирование запросов/ошибок и `faulthandler` для диагностики падений native DLL.

### 1.3 Последний фикс (по 500 из MaxIm)
- Причина: необработанный `OSError (access violation)` внутри `StartMeasurement_DynBitDepth`.
- Решение:
  - в `sdk_adapter.py` добавлен `try/except OSError` на dyn-вызовах;
  - при падении dyn-пути используется fallback на классический `StartMeasurement`/`GetMeasurementData`;
  - ошибки переводятся в управляемый `SdkError`, чтобы не ронять сервис необработанным исключением.
- Дополнительно стабилизирован `/camera/cooling/status` на transient `camera busy` (кешированный статус вместо 500).

## 2) Как система работает сейчас

Архитектура:
- ASCOM клиент (MaxIm DL/другой ASCOM софт) -> C# драйвер (`ASCOM.ProjectR1.Camera`) ->
  Python API (`FastAPI`, `http://127.0.0.1:3037/api/v1`) -> `greateyes.dll` (native SDK) -> камера.

Поток получения изображения:
1. ASCOM вызывает `StartExposure`.
2. Python runtime стартует экспозицию через SDK.
3. Мониторинг завершения переводит задачу в `image_ready`.
4. C# драйвер запрашивает `GET /camera/images/latest/raw`.
5. Бинарный `uint16` кадр собирается в `ImageArray` без JSON-ограничений.

Устойчивость:
- На уровне SDK есть lock и retry/backoff.
- На API-уровне критические SDK-сбои маппятся в управляемые ошибки.
- Для MaxIm-совместимости исключены тяжелые JSON-сценарии передачи кадра.

## 4) Инструкция по установке и развертыванию (чистый/повторный запуск)

## 4.1 Требования
- Windows 10/11.
- Python 3.11+.
- Node.js + npm (для UI/тестов UI).
- Visual Studio Build Tools 2022 (MSBuild).
- .NET Framework 4.7.2 Developer/Targeting Pack.
- ASCOM Platform 6.6+.
- Установленный вендорский пакет `greateyesVision` и доступный `greateyes.dll`.

## 4.2 Первичная настройка
1. Запустить PowerShell **от имени администратора**.
2. Перейти в корень проекта:
   - `cd c:\Project\Project_R1`
3. Выполнить деплой:
   - `.\07_Deployment_and_Ops\deploy_camera_pc.ps1`
4. Проверить/заполнить `02_Python_Camera_Service\.env`:
   - `SDK_MODE=native`
   - `SDK_DLL_PATH=<полный путь к greateyes.dll>`
   - `SDK_CAMERA_ADDRESS=<IP камеры>`
   - `SDK_CAMERA_INTERFACE`, `SDK_CAMERA_PORT`
   - `COOLING_HARDWARE_OPTION` и связанные параметры по стенду.
5. Запустить сервис:
   - `.\07_Deployment_and_Ops\start_camera_service.ps1`

## 4.3 Проверка работоспособности
1. Health:
   - `Invoke-RestMethod http://127.0.0.1:3037/api/v1/health`
2. Подключение камеры:
   - `POST /api/v1/camera/connect`
3. Тест экспозиции:
   - `POST /api/v1/camera/exposures`
   - `GET /api/v1/camera/exposures/{id}/status`
4. Проверка кадра:
   - `GET /api/v1/camera/images/latest/raw`
5. Проверка COM (32-bit клиентский контур):
   - `07_Deployment_and_Ops\tools\com32_smoke_test.ps1`
   - `07_Deployment_and_Ops\tools\com32_extended_methods_test.ps1`

## 4.4 Подключение в MaxIm DL
1. Закрыть лишние приложения, которые могут держать камеру/SDK.
2. В MaxIm выбрать ASCOM драйвер `ASCOM.ProjectR1.Camera`.
3. Выполнить connect.
4. Проверить:
   - одиночную экспозицию;
   - abort/stop;
   - cooler on/off и setpoint;
   - получение `ImageArray` (без `maxJsonLength`).

## 4.5 Операционные заметки
- При повторных transient `camera busy`:
  - выполнить disconnect/connect;
  - убедиться, что нет конкурирующего клиента SDK;
  - проверить `02_Python_Camera_Service/logs/camera_service.log`.
- Для формального закрытия эксплуатационного риска нужен длительный hardware-soak.
